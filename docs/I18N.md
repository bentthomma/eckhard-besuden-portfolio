# I18N

The site supports German and English through `js/i18n.js`.

## Source of truth

All UI translation strings live in one place:

- `js/i18n.js`

There should not be a second translation source in HTML comments, docs, or ad-hoc JS fragments.

## Supported attribute patterns

- `data-i18n`
  Replaces element text content.
- `data-i18n-placeholder`
  Replaces input placeholder text.
- `data-i18n-aria`
  Replaces `aria-label`.

## Runtime behavior

`window.i18n.setLanguage(lang)` does four things:

1. Replaces visible UI copy in the DOM.
2. Updates `<html lang>`.
3. Updates document title and meta description.
4. Dispatches a `langchange` event for any controller that needs to refresh derived UI.

## Current scope

The system covers:

- navigation
- homepage copy
- gallery copy
- overlay labels
- legal modal titles
- accessibility labels
- bid form labels

## Editing rules

- Add new keys to both `de` and `en`.
- Prefer Unicode escapes in JS strings when non-ASCII characters are needed.
- Keep wording short in UI labels and more natural in paragraph copy.
- Do not mix translated and hard-coded visible UI for the same component.

## Related consumers

- `js/nav.js`
  Menu labels and modal labels.
- `js/home-carousel.js`
  Plaque labels and autoplay labels.
- `js/gallery.js`
  Search placeholder and gallery page meta copy.
- `js/artwork-overlay.js`
  Detail labels and bid flow copy.
